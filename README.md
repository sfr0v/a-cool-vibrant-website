# Azure Bay — минималистичный лендинг курорта

Дроновое видео проигрывается **один раз как стартовая интро-анимация**,
затем застывает на **последнем кадре — он и становится фоном сайта**
(слой с видео уходит под контент, поверх кадра — лёгкое плоское затемнение
для читаемости). Дальше — обычная страница: о курорте, удобства,
**магазин и бронь на белом фоне**, чёрный футер.

Палитра строго из трёх цветов: **белый** (текст на кадре; фон магазина
и брони), **чёрный** (кнопки, детали, шапка при прокрутке, футер),
**синий** `#002FA7` (запасной фон, если видео недоступно).
Без градиентов, теней и скруглений.
Иконки — из открытых наборов [Feather](https://feathericons.com) (MIT)
и [Lucide](https://lucide.dev) (ISC), встроены inline как SVG-спрайт:
ноль внешних запросов. Внешних библиотек нет вообще — чистый JS.

## Структура

```
flybysite/
├── index.html         — разметка + SVG-спрайт иконок
├── css/style.css      — чистый CSS, палитра в :root
├── js/main.js         — интро, появление секций, корзина, форма брони
└── assets/flyover.mp4 — видео интро
```

## Запуск

```bash
npx http-server -p 4173 -c-1
```

и открыть <http://localhost:4173>.

## Как подключить своё видео

Положите файл в `assets/` и укажите путь в **data-src** (не в `src` —
загрузкой управляет JS: при экономии трафика, сети 2G или
`prefers-reduced-motion` интро пропускается и видео не грузится вовсе):

```html
<video id="intro-video" class="intro__video"
       data-src="assets/my-flight.mp4"
       muted playsinline preload="none"></video>
```

`muted` и `playsinline` обязательны — без них мобильные браузеры
запрещают автозапуск.

Рекомендуемая подготовка файла (лёгкий старт, без звука):

```bash
ffmpeg -i исходник.mp4 -an -vf "scale=1920:-2" -c:v libx264 -pix_fmt yuv420p -crf 23 -movflags +faststart assets/flyover.mp4
```

ffmpeg на Windows: `winget install Gyan.FFmpeg`.

## Настройки (`CONFIG` в начале js/main.js)

| Параметр                 | Значение | Что делает                                        |
|--------------------------|----------|---------------------------------------------------|
| `INTRO_MAX_WAIT`         | `4000`   | мс: видео не началось за это время → интро пропускается |
| `INTRO_HARD_CAP`         | `20000`  | мс: абсолютный потолок длительности интро         |
| `INTRO_ONCE_PER_SESSION` | `false`  | `true` → интро показывается один раз за вкладку   |

## Палитра

Все цвета — в `:root` в `css/style.css`:

```css
--c-blue:  #002fa7;  /* фон */
--c-white: #ffffff;  /* текст */
--c-black: #0a0a0a;  /* кнопки, детали */
```

## Магазин и бронь

- Товар — это `<li class="product">` с `data-name` и `data-price`;
  чтобы добавить товар, скопируйте карточку и поменяйте данные.
  Корзина (количество + сумма) хранится в `sessionStorage`.
- Форма брони валидирует даты (выезд позже заезда, не в прошлом) и email.
  Реальная отправка подключается в обработчике `submit`
  в `js/main.js` (помечено комментарием «ДЕМО»).

## Фон-кадр и деградация

- После интро видео ставится на паузу на последнем кадре и остаётся
  фиксированным фоном (`z-index: -1`). Если нажали «Пропустить» —
  ролик просто перематывается на последний кадр: фон одинаков у всех.
- Степень затемнения кадра — alpha в правиле `.intro::after`
  в `css/style.css`.
- `prefers-reduced-motion` / Save-Data / 2G → интро и анимации отключены,
  видео не грузится, фон — запасной синий.
- Автоплей запрещён или видео сломалось → то же: синий фон без интро.
- JS выключен → интро скрыто через `<noscript>`, контент виден.
